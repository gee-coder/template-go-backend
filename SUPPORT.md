# 支持说明

## 1. 仓库定位

本仓库是创业技术底座中的 Go 后端模板，默认用于快速启动认证、RBAC、菜单管理与公开联系表单等通用能力。

该仓库提供的是模板级支持，不承诺生产 SLA。若你是基于本模板二次开发的项目方，建议结合自己的业务现状补充团队内部支持流程。

## 2. 提问前建议先检查

- [README.md](./README.md)
- [CONTRIBUTING.md](./CONTRIBUTING.md)
- [SECURITY.md](./SECURITY.md)
- [RELEASE.md](./RELEASE.md)
- `configs/*.yaml`、`.env.example` 与 `docker-compose.yml`

## 3. 适合提交公开 Issue 的场景

- 模板代码存在可复现的缺陷
- 文档、示例配置或脚本与实际行为不一致
- 某项能力缺失，且适合作为通用模板能力沉淀
- 依赖升级后出现兼容性回归

## 4. 不建议提交公开 Issue 的场景

- 安全漏洞、敏感信息泄露、权限绕过与攻击路径讨论
- 带有公司内部地址、账号、密钥、生产日志的排障信息
- 明显属于业务定制开发而非模板共性问题的需求

以上内容请改走 [SECURITY.md](./SECURITY.md) 中的流程，或在脱敏后重新整理问题。

## 5. 提问时建议包含的信息

- 使用环境：Windows / macOS / Linux、Go 版本、MySQL / Redis 版本
- 启动方式：本机运行或 Docker Compose
- 操作步骤：从空库到出现问题的最短复现路径
- 实际结果与期望结果
- 脱敏后的日志、报错栈、配置片段

## 6. 维护者处理优先级

- `bug report`：优先处理可复现且影响模板主流程的问题
- `feature request`：优先评估能否沉淀为跨项目复用能力
- 文档问题：通常会和最近一次文档迭代一起处理

## 7. Fork 使用建议

如果你是将本仓库 fork 后用于团队内部项目，建议同步调整以下内容：

- 仓库说明、维护者名单与响应窗口
- 安全反馈邮箱或私密沟通渠道
- 发布节奏与版本号策略
