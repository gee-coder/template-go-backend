# 维护者说明

## 1. 当前维护角色

- 仓库所有者：`gee-coder`
- 默认维护范围：模板结构、通用能力、文档、CI、依赖升级策略

如果你是 fork 后的使用方，建议将本文件中的维护人、评审人与发布责任人替换为自己的团队成员。

## 2. 维护边界

以下改动建议至少经过一名维护者确认后再合并：

- 认证、权限、刷新令牌与中间件行为变更
- 公共 API 字段、响应结构、错误码语义变更
- 核心依赖的大版本升级
- Docker、CI、脚本与项目结构调整

## 3. 合并标准

合并前建议满足以下条件：

- CI 通过
- 涉及行为变更时同步补充测试或说明验证方式
- 文档、示例配置、Swagger 注释保持同步
- 不引入无明确收益的破坏性变更

## 4. 分支与版本建议

- 默认稳定分支：`master`
- 功能分支建议：`feat/*`
- 修复分支建议：`fix/*`
- 发布标签建议：`vX.Y.Z`

## 5. 发布责任

发布前由维护者确认：

- `CHANGELOG.md` 已更新
- `RELEASE.md` 中的检查项已完成
- 需要同步更新的文档与示例配置已落地
